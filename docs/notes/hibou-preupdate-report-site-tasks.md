# Hibou 更新前診断レポート 掲載メモ

関連仕様（SunabaEye 側）:
- https://github.com/marushu/sunabaeye/blob/main/docs/notes/hibou-preupdate-report.md

## 目的
hibou-web.com 上で「WordPress / PHP 更新前診断レポート」を、実体のあるサービスとして見せられるようにする。

## 前提
- レポートの診断・要約エンジンは SunabaEye を利用する
- 表向きは Hibou のサービスとして提供する
- 先に売るのは自動化SaaSではなく、依頼しやすい実務サービス

## 対応したい内容
### 1. サービスページへの反映
候補ページ:
- `/before-update`
- 必要に応じて `/wordpress-update` からも導線を張る

盛り込みたい内容:
- 更新してよいかを、先に整理するサービスであること
- PHP / WordPress 更新時の影響を事前に見える化すること
- 問題点だけでなく、回避案までレポートすること
- 先方で対応するか、Hibou に依頼するかの判断材料になること

### 2. レポートサンプル掲載
見せ方は2段構えにする:
- 一般向けサマリー
- 詳細サンプル

#### 一般向けサマリーに入れたいもの
- 何がわかるか
- Green / Yellow / Red の意味
- どんな人に向いているか
- 修正作業は別途相談できること

#### 詳細サンプルに入れたいもの
- 対象項目
- 判定
- 関連ファイル / テーマ / プラグイン
- 想定される影響
- 回避案
- 対応主体（先方対応可 / Hibou 対応推奨）

### 3. FAQ 追加
候補:
- 調査だけでも依頼できますか
- そのまま修正も依頼できますか
- WordPress 本体だけでなく PHP 更新も見てもらえますか
- 独自テーマでも対象になりますか
- レポートで Green なら安全ですか

### 4. CTA 整理
- まずは調査のみ相談可能
- 修正は必要に応じて別途対応
- 人手不足の制作会社・保守委託先でも頼みやすい導線にする

### 5. JSON-LD / 構造化データ
- Service
- FAQPage

## コピーの方向性メモ
- 更新してよいかを、先に整理します
- PHP / WordPress 更新時の影響を、事前に見える化します
- 問題点だけでなく、回避案までレポートします
- 先方で対応するか、依頼するかの判断材料になります

## やらないこと
- 恐怖訴求
- 過度な断定
- Green = 完全保証 のような見せ方

## LP / サイト実装タスク候補
- `/before-update` に診断レポート導線追加
- レポートサンプルの簡易モック作成
- FAQ 追加
- JSON-LD 追加
- 必要であればトップや `/wordpress-update` から導線整理

## 運用メモ
- サンプルレポートは実在案件そのままではなく、ダミー化して掲載する
- 将来的には SunabaEye ダッシュボードの出力形式と揃える
- 手動サービスとして先に売り、後から自動化へつなげる
